Student Course Portal — Project File Notes
============================================

1. angular.json
   The Angular workspace configuration file — defines projects, build targets
   (build/serve/test), file replacements, asset paths, and budget limits used
   by the Angular CLI to compile and run the app.

2. tsconfig.json
   The root TypeScript configuration shared across the whole workspace —
   sets compiler options (target, strict mode, module resolution) that all
   other tsconfig files (app/spec) extend from.

3. tsconfig.app.json
   Extends tsconfig.json specifically for the application build — lists the
   files/entry points to include (main.ts) and excludes test (*.spec.ts) files.

4. package.json
   Lists the project's npm dependencies (Angular packages, RxJS, etc.),
   devDependencies (CLI, TypeScript, testing tools), and the npm scripts
   (ng serve, ng build, ng test) used to run common CLI commands.

5. src/main.ts
   The application's entry point — it bootstraps the root standalone
   component (App) using bootstrapApplication() and the providers defined
   in app.config.ts (this replaces the older AppModule/platformBrowserDynamic
   bootstrap approach).

6. src/app/app.config.ts (standalone mode equivalent of app.module.ts)
   Defines the application-wide providers (router, zone change detection,
   HTTP client, etc.) passed into bootstrapApplication() — this is the
   standalone-API replacement for the old NgModule-based AppModule.

7. src/app/app.ts (standalone equivalent of app.component.ts)
   The root component of the app — its selector (app-root) is what's placed
   in index.html, and its template (app.html) hosts the <app-header> and
   <router-outlet> that render the rest of the application.
   Note: In this Angular CLI version (v20.x) the default schematic drops the
   ".component" suffix from generated filenames/class names, so what earlier
   Angular versions called app.component.ts/AppComponent is now generated as
   app.ts/App — same role, updated naming convention.

8. src/index.html
   The single real HTML page served to the browser — contains the
   <app-root></app-root> tag where Angular bootstraps and renders the entire
   single-page application.

Additional exploration notes
-----------------------------
- ng serve: compiles the app in memory and serves it via a dev server at
  http://localhost:4200 with live reload. Verified the default Angular
  welcome page loaded successfully before components were added.

- ng build: produces a dist/student-course-portal/browser/ folder. The
  compiled application logic lives in the hashed main-<hash>.js bundle
  (application code), alongside polyfills-<hash>.js (browser polyfills),
  styles-<hash>.css (global styles), and index.html/index.csr.html which
  reference those bundles.

- angular.json budgets (architect > build > configurations > production):
  Under the "budgets" array, entries like:
    { "type": "initial", "maximumWarning": "500kB", "maximumError": "1MB" }
  control the maximum allowed size of the compiled output bundles.
  - maximumWarning: if the bundle exceeds this size, the CLI prints a
    build warning but the build still succeeds.
  - maximumError: if the bundle exceeds this size, the CLI fails the build
    with an error, forcing you to reduce bundle size before shipping.
  These budgets exist to catch bundle-size regressions before they reach
  production.
